# [번역·정리] CausalRAG — Integrating Causal Graphs into Retrieval-Augmented Generation

> **인과 그래프를 검색증강생성에 통합** · Wang et al., Case Western Reserve University · 2025 · [arXiv 2503.19878](https://arxiv.org/abs/2503.19878) · [github.com/Pwnb/CausalRAG](https://github.com/Pwnb/CausalRAG)
> RAG의 검색을 **의미 유사도가 아니라 인과(causal) 관계**로 하자는 접근. (학습용 번역 + ragbench 연결)

---

## 초록 (Abstract) — 전문 번역

LLM은 외부 지식을 통합하는 RAG로 NLP를 혁신했다. 그러나 전통적 RAG는 치명적 한계가 있다: **텍스트 청킹으로 인한 맥락 무결성 붕괴**, 그리고 **검색 시 의미 유사도에 과의존**. 이를 해결하기 위해 **CausalRAG** — 검색 과정에 **인과 그래프**를 통합하는 새 프레임워크 — 를 제안한다. 인과 관계를 **구축·추적**함으로써 CausalRAG는 **맥락 연속성을 보존하고 검색 정밀도를 향상**시켜, 더 정확하고 해석가능한 응답을 낸다. 일반 RAG·그래프 기반 RAG와 비교해 여러 지표에서 우월함을 입증했다. **검색을 인과 추론에 기반**시키는 것이 지식집약 과제에 유망함을 시사한다.

---

## 1. 서론 — 핵심 문제의식

- LLM은 사전학습 지식에 의존 → **동적으로 갱신되는 외부 정보** 통합·추론에 한계(특히 학술연구 같은 지식집약 도메인). RAG가 이를 완화.
- 최근 RAG 연구 두 방향: ① 검색 효율·통합 개선 ② **외부 지식 표현·다단계 활용(그래프 RAG가 대세)**.
- 그럼에도 기존 RAG의 3대 한계 → LLM이 **"그럴듯하지만 얕고 핵심 디테일이 빠진" 응답**을 내는 주원인.
- **CausalRAG**: 인과 그래프로 **검색 정확도·추론 성능** 향상. 일반/그래프 RAG와 달리 **인과 관계를 명시적으로 식별** → 맥락 일관성 유지 + 인과 의존성 포착. 검색문서가 **관련 있으면서 인과적으로 근거 있음(causally grounded)** 을 보장 → 더 풍부·인과적으로 상세한 응답. 환각↓·충실성↑.

## 2. 왜 일반 RAG는 정확한 응답에 실패하는가 (핵심 분석)

### 3대 한계 (Figure 1a)
1. **청킹이 언어적 일관성 붕괴** — 텍스트를 최소 단위로 자르면 원문의 자연스러운 언어·논리 연결이 끊김. 맥락 무결성 위해 복원 메커니즘 필요. (*우리도 청크 트레이드오프를 겪음*)
2. **의미 검색의 편향** — 벡터 DB에서 질의와 **의미적으로 가장 가까운** 문서를 검색. 하지만 답에 필요한 핵심 정보는 **의미가 아니라 인과로 관련**된 경우가 많음.
   - 🍼🍺 **고전 예시**: "기저귀와 맥주" — 의미적으론 안 비슷하지만 현실에서 **인과 연결**(마트에서 함께 팔림). 의미 유사도는 **맥락상 무관하지만 표면상 관련된** 정보를 검색함.
3. **생성에서 편향 증폭** — 관련 맥락을 검색해도 정확한 응답을 보장 못 함.

### ★ Recall–Precision 관점 (우리 프로젝트와 직결!)
- **Context Recall** = 검색된 게 정답 참조집합 `R`에 속하는 비율:
```
Context Recall = Σ 𝟙(C_i ∈ R) / |R|
```
- **Context Precision** = 검색된 것 중 **인과적으로 질의에 관련된** 비율:
```
Context Precision = Σ 𝟙_ℚ(C_i ∈ R) / Σ 𝟙(C_i ∈ R)     (𝟙_ℚ = 인과 관련이면 1)
```
- **일반 RAG = recall 높고 precision 낮음** — 의미 유사로 관련 문서를 많이 가져오지만(recall↑), **진짜 인과 관련(precision)** 은 낮음. 예: "이 논문은 AI를 어떻게 정의하나?" → 의미 비슷한 모든 정의를 가져오지만 정작 저자 정의만 관련.
- **낮은 precision = 체계적 편향** → 검색 맥락의 정확도·신뢰도 희석.

### GraphRAG 재고 (Figure 1b)
- 그래프 RAG는 **검색 전 그래프 정보 요약·랭킹** → **precision 향상**. 그러나:
  - 요약이 무관 정보를 완전히 못 걸러 **부분적 해결**에 그침.
  - **서브그래프 요약 의존이 recall을 해칠 수 있음** — 그래프 중심성은 낮지만 인과적으론 관련된 걸 누락. → **precision↑ / recall 소폭↓ 트레이드오프**.

> 💡 *이 "그래프는 precision 올리나 recall 희생" 분석은 우리 실측과 정확히 일치. 우리는 여기서 **그래프+직접벡터 하이브리드**로 recall까지 회복(0.45→0.83)했음.*

## 3. 방법 (CausalRAG 아키텍처, Figure 2)

### 3.1 색인 (오프라인)
- 업로드 문서를 **텍스트 기반 그래프**로 구축(LangChain 방식 — LLM이 엔티티/개념=노드, 관계=엣지 추출). 그래프를 임베딩해 벡터 DB 저장. 질의도 임베딩. **질의 시간과 독립**(빠른 추론).

### 3.2 인과 경로 발견·추정 (질의 시)
1. **초기 노드 k개**: 질의 임베딩과 거리가 가장 가까운 그래프 노드 k개 선택(질의와 직접 관련). k=튜너블(크면 더 많이·비용↑).
2. **노드 확장 s단계**: 기본 그래프 엣지를 따라 step size `s`만큼 확장 → **인과·관계 연결 보존**하며 맥락 확대(recall 유지). s=확장 깊이.
3. **인과 경로 추정**: 수집된 노드·엣지에서 **LLM이 인과 경로를 식별·추정** → 정제된 **인과 그래프** + **인과 요약(causal summary) 보고서** 생성.
4. 인과 요약 = **근거 있는(grounded) 검색 컨텍스트** → 질의와 결합해 LLM 생성.
- 인과 그래프의 두 역할: ① 전통 검색이 놓치는 **인과 관련 정보 보존** ② precision 향상.

---

## 4. 평가
- 다양한 도메인·컨텍스트 길이에서 일반 RAG·그래프 RAG와 비교. 3지표: **answer faithfulness, context recall, context precision** (Ragas 프레임워크).
- CausalRAG가 여러 지표에서 우월. 사례연구·파라미터 분석 포함.

## 💡 학습 포인트 · 우리 프로젝트(ragbench) 연결

| CausalRAG 논문 | ragbench에서 |
| --- | --- |
| **recall-precision 관점** | **우리 핵심 렌즈** — 그래프는 precision 성향/recall 희생, 벡터는 recall |
| "의미 유사 ≠ 인과 관련" | 우리 재랭커 이슈("의미 비슷한 잡다한 청크가 정답을 밀어냄")와 상통 |
| 그래프는 precision↑ recall↓ | **우리 실측과 일치** → 우리는 하이브리드로 recall 회복 |
| 초기노드 k + s단계 확장 | 우리 그래프 검색(진입노드+이웃확장)과 유사 구조 |
| 청킹이 맥락 붕괴 | 우리 청크 트레이드오프 경험 |
| 인과 요약을 컨텍스트로 | 우리 L5 커뮤니티 요약 주입과 발상 유사(요약을 검색 컨텍스트로) |

**우리 결론과의 관계**: CausalRAG는 "의미 유사 대신 **인과 관련**으로 검색하면 precision↑"을 주장. **우리는 다른 각도**로 같은 문제(precision)를 해결 — **그래프 검색 + 직접 벡터검색 융합**으로 정밀도를 평면 수준까지. 즉 CausalRAG는 "검색 신호를 인과로 교체", 우리는 "검색 신호를 융합(그래프+벡터)". 둘 다 **"단일 검색 신호의 한계"** 를 공략한다는 점에서 통한다.

> 세 논문 관통 주제: **RAG(원조: 검색+생성 결합) → GraphRAG(검색을 그래프+커뮤니티로) → CausalRAG(검색을 인과로).** 모두 **"어떻게 더 정확히 검색하느냐"** 의 진화. 우리 프로젝트 결론("검색층이 성능을 좌우, 단일 신호는 약하다")이 이 흐름의 핵심을 관통.
