# [번역·정리] RAG — Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks

> **지식집약 NLP 과제를 위한 검색증강생성** · Lewis et al., Facebook AI / UCL / NYU · NeurIPS 2020 · [arXiv 2005.11401](https://arxiv.org/abs/2005.11401)
> RAG의 원조 논문. 이후 GraphRAG·LightRAG·HippoRAG 등 모든 검색증강 기법의 토대. (학습용 번역 — 핵심 섹션 + 용어 설명 + 우리 프로젝트 연결)

---

## 초록 (Abstract) — 전문 번역

대형 사전학습 언어모델은 사실 지식을 **파라미터 안에** 저장하며, 다운스트림 NLP 과제에 파인튜닝하면 SOTA를 낸다. 하지만 **지식에 정확히 접근·조작하는 능력은 여전히 제한적**이라, 지식집약 과제에선 과제특화 아키텍처에 성능이 뒤진다. 또한 **결정 근거 제시(provenance)와 세계 지식 갱신**은 미해결 문제다.

명시적 **비(非)파라미터 메모리**에 미분가능하게 접근하는 사전학습 모델은 지금까지 추출형(extractive) 과제에서만 연구됐다. 본 논문은 **검색증강생성(RAG)** — **사전학습 파라미터 메모리 + 비파라미터 메모리**를 결합해 언어를 생성하는 범용 파인튜닝 레시피 — 를 탐구한다.

RAG는 파라미터 메모리 = **사전학습 seq2seq 모델**, 비파라미터 메모리 = **위키피디아의 밀집 벡터 인덱스**(사전학습 신경 검색기로 접근)로 둔다. 두 형태를 비교한다: ① 생성 전체 구간에 **같은 검색 문단**을 쓰는 것, ② 토큰마다 **다른 문단**을 쓸 수 있는 것. 광범위한 지식집약 과제로 파인튜닝·평가해, **3개 오픈도메인 QA에서 SOTA**를 세우고 파라미터-only seq2seq·과제특화 검색-추출 아키텍처를 능가했다. 생성 과제에선 RAG가 **SOTA 파라미터-only 베이스라인보다 더 구체적·다양·사실적인** 언어를 생성했다.

## 1. 서론 핵심

- **문제**: 사전학습 LM은 외부 메모리 없이 지식을 파라미터에 암묵 저장 → ① 메모리 확장·수정 어려움 ② 예측 근거(insight) 제시 어려움 ③ **환각(hallucination)** 발생.
- **해결**: **하이브리드 메모리** — 파라미터(생성) + 비파라미터(검색). 지식을 직접 수정·확장·검사·해석 가능.
- 기존 REALM·ORQA는 오픈도메인 **추출형 QA**만 다룸. 본 논문은 이걸 **"NLP의 일꾼" seq2seq 모델**로 확장.

---

## 2. 방법 (Methods) — 핵심

입력 `x`로 문서 `z`를 검색해, 생성 시 **추가 컨텍스트**로 사용. 두 구성요소:
- **검색기(retriever) `p_η(z|x)`** — 질의 `x`에 대한 문단 top-K 분포
- **생성기(generator) `p_θ(y_i|x, z, y_{1:i-1})`** — 이전 토큰·입력·검색문단 기반 현재 토큰 생성

검색 문서 `z`를 **잠재변수(latent variable)** 로 두고, 두 방식으로 주변화(marginalize):

### 2.1 두 모델
**RAG-Sequence** — **전체 시퀀스**에 **같은 문서 하나** 사용:
```
p(y|x) ≈ Σ_{z∈top-k}  p_η(z|x) · Π_i p_θ(y_i|x, z, y_{1:i-1})
```
**RAG-Token** — **토큰마다 다른 문서** 사용 가능(여러 문서 내용 조합):
```
p(y|x) ≈ Π_i  Σ_{z∈top-k}  p_η(z|x) · p_θ(y_i|x, z, y_{1:i-1})
```

### 2.2 검색기: DPR (Dense Passage Retriever)
**바이-인코더(bi-encoder)** 구조:
```
p_η(z|x) ∝ exp( d(z)ᵀ q(x) )    d(z)=BERT_d(z),  q(x)=BERT_q(x)
```
- 문서·질의를 각각 BERT로 밀집 벡터화 → 내적 유사도. top-k는 **MIPS**(최대내적검색)로 sub-linear 시간에.
- 문서 인덱스 = **비파라미터 메모리**. TriviaQA·NQ 정답 포함 문서를 검색하도록 사전학습된 DPR로 초기화.

### 2.3 생성기: BART
- **BART-large(400M)** — 사전학습 seq2seq. 입력 `x`와 검색내용 `z`를 **단순 연결(concatenate)** 해 생성. 파라미터 `θ` = **파라미터 메모리**.

### 2.4 학습
- 검색기·생성기를 **어떤 문서를 검색해야 하는지 직접 감독 없이** 함께 학습. 목표: 각 타깃의 **음의 주변 로그가능도** 최소화(Adam).
- **문서 인코더(BERT_d)는 고정**, 질의 인코더(BERT_q)·BART만 파인튜닝(문서 인덱스 재구축 비용 회피 — REALM과 차이점).

### 2.5 디코딩
- **RAG-Token**: 표준 autoregressive 빔서치.
- **RAG-Sequence**: 문서별 빔서치 후 가설 점수 합산("Thorough Decoding") / 효율판 "Fast Decoding".

## 3. 실험 & 결과 핵심

- **비파라미터 소스**: 위키피디아 2018.12 덤프, 100단어 청크 **2,100만 개**, FAISS MIPS 인덱스. 학습 시 top k∈{5,10}.
- **과제**: 오픈도메인 QA(NQ·TriviaQA·WebQuestions·CuratedTrec), 추상형 QA(MS-MARCO), Jeopardy 질문 생성, 사실검증(FEVER).

**결과**
- **오픈도메인 QA 4종 모두 SOTA**(TQA는 T5 비교분할 한정). "closed-book(파라미터)"의 생성 유연성 + "open-book(검색)"의 성능을 결합. **재랭커·추출 리더 없이도** SOTA — *우리 프로젝트의 "검색이 답을 골라온다"와 직결*.
- **정답이 검색문서에 없어도** 생성 가능: NQ에서 그런 경우 11.8% 정답(추출모델은 0%).
- 추상형 QA(MS-MARCO): RAG-Seq가 BART보다 BLEU +2.6·ROUGE-L +2.6. **환각 적고 더 사실적**.
- Jeopardy 생성: 사람 평가에서 RAG가 BART보다 **더 사실적(42.7% vs 7.1%)·더 구체적**. RAG-Token이 **여러 문서 내용을 조합**(예: "The Sun Also Rises" + "A Farewell to Arms").
- **파라미터·비파라미터 메모리의 협업**: 비파라미터가 생성을 유도하고, 파라미터가 세부 지식을 채움.
- FEVER 사실검증: 검색 감독 없이도 SOTA의 4.3% 이내.

## 💡 학습 포인트 · 우리 프로젝트(ragbench) 연결

| RAG 논문 개념 | ragbench에서 |
| --- | --- |
| 검색기(DPR 벡터) + 생성기(BART) | `standard`(벡터검색) + LLM 생성 = 이 구조 |
| top-k 문서를 **함께 넣어 생성** | 우리 파이프라인 "컨텍스트 동시 전달 → LLM" (ARCHITECTURE.md) |
| 잠재변수 주변화(RAG-Sequence/Token) | 우리는 단순화해 top-k를 한 프롬프트로 투입(현대 LLM RAG 방식) |
| "재랭커·추출 리더 없이 SOTA" | 우리 결론 "검색층(query→청크)이 정밀도의 핵심"과 상통 |
| 비파라미터 메모리 갱신으로 지식 수정 | GraphRAG·그래프 저장소가 이걸 구조화 |

**한 줄 요약**: RAG = **파라미터(생성 LLM) + 비파라미터(검색 인덱스)** 를 결합해, 지식을 외부에서 검색해 넣어 환각↓·사실성↑·갱신가능. 이후 모든 검색증강 기법(GraphRAG 등)이 이 위에 검색 방식을 고도화한 것.

> 다음: GraphRAG(2404.16130) — 이 "검색"을 **지식그래프+커뮤니티 요약**으로 확장. / CausalRAG(2503.19878) — **인과 그래프**로 확장.
